▶ OPC自习室

■"Copilot要杀死低代码？"

我们试了三个月，发现被骗了 _

有人说2026年低代码已死，我们偏不信邪。一场真实的"人机大战"后，CTO老张红着眼圈说：差点把家底都赔进去。

01 那个让CTO失眠的深夜

// 灵魂拷问：AI都能自动写代码了，低代码平台是不是该扔了？

凌晨两点，老张盯着屏幕上的代码，手里的咖啡早就凉了。

他是某制造企业的技术负责人，最近被一个"灵魂拷问"折磨得睡不着觉——

"AI都能自动写代码了，我们花百万买的低代码平台，是不是该扔了？"

这句话来自CEO。上周的周会上，老板把一篇爆款文章摔在桌上，标题赫然写着：《Copilot来了，低代码进入倒计时》。

老张没吭声。他知道，公司里至少有三个技术骨干已经在偷偷用Cursor写代码了，效率确实高。更可怕的是，上周一个实习生用ChatGPT加v0，两个小时搭出来一个数据看板，前端组的老王干了三天。

那晚，老张做了一个决定：用一个月，让AI代码生成和低代码平台正面打一仗。

02 第一回合：AI的"高光时刻"

// 前三天，AI队碾压式领先

老张挑了一个中等难度的任务——给生产车间做一个设备报修表单系统。

他分了两队：

A队：用Copilot + Cursor，纯AI生成代码，前端React，后端Node.js，数据库PostgreSQL。带队的是一年经验的小李，AI用得溜。

B队：用公司现有的低代码平台（含流程引擎和表单设计器），带队的是干了八年开发的王哥，低代码熟手。

前三天，AI队碾压式领先。

小李对着Cursor说了一句话："帮我做一个设备报修表单，包含设备编号、故障描述、紧急程度三选一，提交后存数据库。"十秒钟，表单代码出来了。

三天后，AI队已经跑通了基本流程。小李兴奋地在群里发截图："看，全程没写一行代码！"

低代码队呢？王哥还在拖拽组件、配置数据源。不是慢，是因为要对接公司现有的ERP系统，读设备台账、写工单状态回传，低代码那边需要先配接口。

第一回合，AI队完胜。老张心里咯噔一下：难道预算真该砍？

03 AI写出来的代码，像"没有工地的建筑"

// 转折：AI猜来猜去，Bug越修越多

第七天，问题来了。

测试组报了一个Bug：紧急工单提交后，维修组收到的通知有时延迟十分钟。

小李开始调试。他让AI帮忙定位问题，AI给了三个可能的解释，但都是泛泛而谈——连接池、异步队列、网络抖动。

[!] 问题爆发

"到底是什么？"小李一遍遍问，AI一遍遍换着花样猜。他花了整整一天，最后发现自己没加事务控制——当数据库写入和消息通知同时失败时，工单状态变成"待处理"，但维修组没收到。系统出现了"幽灵工单"。

更麻烦的是，这个Bug要改，涉及六个文件、三种语言。AI每次只改一小块，改完又冒出新的问题。

老张后来跟朋友形容：

"就像你找了个施工队，他们半天给你搭好了毛坯房，但你一住进去，发现厕所漏水、电路跳闸、窗户关不严。你让施工队来修，他们来了，这儿敲一下，那儿补一下，结果厨房又开始漏水了。"

那边，王哥的低代码队呢？第十天的时候，他开始用平台的"流程引擎"画工单流转图，拖了几个节点，配了条件分支。触发延迟？不存在的，因为平台的异步消息机制是封装好的，从来没出过问题。

第十五天，AI队还在修Bug。小李已经换了三个不同的AI提示词，试了"重写整个模块"，结果把之前好的功能也搞坏了。

老张第一次意识到：AI生成的代码，像一串漂亮但脆弱的积木；而低代码平台，是那个告诉你"这块积木必须放这里才能站稳"的图纸。

04 权限和合规，AI根本不懂

// 第三周，安全审计来了，AI队彻底崩溃

第三周，更严重的事发生了。

公司安全审计突然要求：所有工单系统必须实现"角色分级审批"——部门主管只能看本部门工单，财务只能看涉及备件采购的工单，而且所有操作日志必须保留六个月。

[!] AI队崩溃

小李让AI生成RBAC代码。AI确实生成了，但用的是网上最常见的一套开源方案。问题是，这套方案跟公司现有的AD域认证不兼容。改？AI不知道AD域是什么。小李只能自己翻文档、写适配层，一写就是三天。

更崩溃的是，安全审计要求"多租户数据隔离"，AI生成的SQL语句里完全没考虑租户ID过滤。小李手动加了几十个 WHERE tenant_id = ?，但还是漏了两个查询接口，差点酿成数据泄露。

老张后怕："如果真上线了，我这CTO就不用干了。"

反观低代码平台，王哥点开"角色权限管理"模块，勾选了几个选项，配置了数据行级过滤规则，五分钟搞定。日志审计？平台自带，一键开启。

AI像一匹野马，跑得快，但不认路。低代码是那个缰绳和地图。

05 最后的真相

// 不是谁取代谁，是AI负责"浪漫"，低代码负责"落地"

一个月后的复盘会上，老张放了张PPT，上面只有一行大字：

"AI大模型 × 低代码平台 = 未来的开发范式"

他解释了为什么不是"+"而是"×"：

AI负责生成：快速出原型、写基础CRUD代码、生成测试用例、翻译自然语言需求。它不眠不休，效率是人力的十倍。

低代码负责约束：提供权限、流程、集成、安全、日志等企业级"骨架"。AI生成的代码像血肉，但没有骨架，就是一摊肉泥。

最后，老张给出了公司的新方案：

1. 业务人员用自然语言对AI描述需求
2. AI生成低代码平台可导入的"应用描述文件"
3. 开发人员在低代码平台中微调、配置权限、对接内部系统
4. 上线

这套流程让一个中等复杂度应用的开发周期从三周缩短到五天。
